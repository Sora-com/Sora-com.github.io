Detailed Description

Módulo que contiene las funciones de guardado y carga de partidas desde ficheros de texto.

Este módulo se encarga de leer y posiblemente guardar la información necesaria del juego (como los barcos) desde archivos.

Author
Akira

◆ CargarPartida()

void CargarPartida ( Config * config)

Carga una partida previamente guardada desde un archivo.

Esta función lee los datos de un archivo de guardado y carga toda la configuración del juego, incluyendo escala, número de barcos, turno, repetición de turno, empate, datos de los jugadores, barcos, disparos realizados y tableros de flota.

Parameters
configPuntero a la estructura Config donde se almacenarán los datos cargados.

El archivo de guardado debe seguir el formato esperado:

  • Datos generales de la partida
  • Información de cada jugador (nombre, disparos, tipo de disparo, ganador)
  • Datos internos de búsqueda de barcos de la CPU
  • Lista de tipos de barcos
  • Coordenadas de disparos no realizados
  • Estado de los tableros de cada jugador

Si el archivo no puede abrirse, se imprime un mensaje de error.

◆ GuardarPartida()

void GuardarPartida ( Config * config)

Guarda el estado actual de la partida en un archivo de texto.

Esta función guarda todos los datos necesarios para continuar la partida más adelante, incluyendo configuraciones generales, datos de los jugadores, tipos de barcos utilizados, coordenadas no disparadas, y los tableros de ambos jugadores.

Si la escala de los tableros es menor que 5, se considerará que no se han introducido datos suficientes y se mostrará un mensaje de error. También se asegura que los tableros hayan sido creados antes de guardar.

Parameters
configPuntero a la estructura Config que contiene toda la información de la partida.
Precondition
config debe estar correctamente inicializado y contener todos los datos necesarios para la partida.
Postcondition
Se genera o actualiza un archivo de guardado definido por NOMBRE_FICHERO_GUARDADO.
Warning
Si ocurre un error al abrir el archivo, se muestra un mensaje, pero no se termina la ejecución.

◆ LeerBarcos()

int LeerBarcos ( Barco ** v_barcos,
int * cantBarcos )

Lee los datos de los barcos desde un fichero de texto y los almacena dinámicamente.

Abre el archivo "Barcos.txt", lee los datos de cada barco (nombre, id y longitud) y los guarda en un array dinámico de estructuras tipo Barco.

Parameters
[out]v_barcosPuntero al array dinámico de barcos. Se reserva memoria dentro de la función.
[out]cantBarcosPuntero a entero donde se guarda la cantidad de barcos leídos.
Returns
int Devuelve 0 si la operación fue exitosa, o -1 si hubo un error al leer o al reservar memoria.
Precondition
El archivo "Barcos.txt" debe existir y seguir el formato correcto: nombre-id-longitud por línea.
Postcondition
Si se lee correctamente, v_barcos contendrá todos los barcos leídos y cantBarcos indicará su número.

◆ ReservaPreviaCargarPartida()

void ReservaPreviaCargarPartida ( Config * config)

Reserva o vuelve a reservar la memoria necesaria para continuar una partida previamente guardada.

Esta función se encarga de realizar las reservas dinámicas de memoria para los distintos elementos de la configuración de una partida: barcos, coordenadas no disparadas de los jugadores y los tableros de juego. Dependiendo de si las estructuras ya estaban reservadas previamente, se usa malloc o realloc.

Parameters
configPuntero a la estructura Config que contiene todos los datos de la partida.
  • Reserva (o realoca) memoria para:
    • config->BarcosJuego: array de estructuras Barco.
    • config->Jugador1.coordNoDisparadas y config->Jugador2.coordNoDisparadas: arrays de coordenadas.
    • config->Tablero1 y config->Tablero2: matrices de enteros que representan los tableros.
  • Se maneja el error de memoria en cada reserva y se imprime un mensaje en caso de fallo.
  • Actualiza config->tablerosConfigurados a 2 una vez que ambos tableros han sido reservados.
Warning
Si ocurre un error durante alguna reserva, se muestra un mensaje de error, pero no se libera memoria ya reservada previamente.
